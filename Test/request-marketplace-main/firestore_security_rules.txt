# Firestore Notifications Index Issue

The notification system is showing a count but empty list because of a missing Firestore index.

## Problem:
The query in NotificationService.getUserNotifications() requires a composite index:
- Query: notifications where recipientId==<uid> order by -createdAt, -__name__
- Error: "The query requires an index"

## Solution Applied:
1. Modified getUserNotifications() to remove orderBy from Firestore query
2. Sort results in memory instead: `notifications.sort((a, b) => b.createdAt.compareTo(a.createdAt))`

## Proper Solution (for production):
Create the composite index in Firebase Console:
Collection: notifications
Fields:
- recipientId (Ascending)  
- createdAt (Descending)
- __name__ (Descending)
